Split the browser into independent widgets:
widgets/
browser/
breadcrumbs.rs
file_tree.rs
file_row.rs

file_details.rs

ui.separator();
ui.add(FileTree::new(...));
ui.separator();
ui.add(FileDetailsView::new(...));
No rendering logic should remain inside render_browser_screen() .

Password handling
Improve password security:
keep String only while editing in TextEdit
never clone passwords
use std::mem::take() on submit
immediately convert to Zeroizing<String>
pass Zeroizing<String> through the encryption/decryption pipeline
derive keys from &str or &Zeroizing<String>
clear the UI buffer with zeroize() on cancel/close
drop the password immediately after key derivation
Goals
Produce idiomatic Rust and egui code that is modular, readable, avoids borrow-checker workarounds,
minimizes duplication, and preserves all existing functionality
