Это очень хорошо ложится на архитектуру с единым enum событий, а не на глобальный EventBus.

Я бы сделал так

Создаешь общий enum:

pub enum AppEvent {
    NavigateTo(String),

    SelectFile(FileEntry),

    OpenFile(FileEntry),

    CloseVault,

    Encrypt(PathBuf),

    Decrypt(PathBuf),

    PasswordEntered(Zeroizing<String>),

    PasswordCancelled,

    Progress(ProgressMessage),
}

И каждый виджет возвращает не свой enum, а AppEvent.

Например:

BreadcrumbBar
    ↓
AppEvent::NavigateTo(...)

FileTree
    ↓
AppEvent::SelectFile(...)

FileTree
    ↓
AppEvent::OpenFile(...)

PasswordModal
    ↓
AppEvent::PasswordEntered(...)

Потом в render_browser_screen:

if let Some(event) = FileTree::new(...)
    .show(ui)
{
    self.handle_event(event);
}

И весь код изменения состояния живет в одном месте:

fn handle_event(&mut self, event: AppEvent) {
    match event {
        AppEvent::NavigateTo(path) => { ... }

        AppEvent::SelectFile(file) => { ... }

        AppEvent::OpenFile(file) => { ... }

        AppEvent::CloseVault => { ... }

        ...
    }
}

Это примерный код. Не обязательно рабочий, нужно будет сделать свою имплементацию.
Что я бы действительно сделал

Разделил события.

enum BrowserEvent

enum PasswordEvent

enum HomeEvent

enum ProgressEvent

И сверху

enum AppEvent {
    Browser(BrowserEvent),

    Password(PasswordEvent),

    Progress(ProgressMessage),

    Home(HomeEvent),
}

Тогда каждый модуль знает только про свои события.

Это очень хорошо масштабируется

Например потом добавится:

SettingsView

SearchView

BookmarksView

TagEditor
